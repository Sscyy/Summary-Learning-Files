# 美团猜你喜欢多模态表征与语义ID体系建设

> 本文档用于简历项目描述，覆盖从多模态表征训练到语义ID生成、再到精排下游应用的完整链路。

---

## 项目概述

在美团首页"猜你喜欢"推荐场景中，传统推荐系统依赖商品原始 Hash ID 作为供给表征，存在三个核心痛点：

1. **语义缺失**：Hash ID 是随机整数，不携带任何语义信息，模型只能依赖行为共现进行"机械记忆"，长尾商品和新品冷启动效果差
2. **模态单一**：商品的图片、标题等丰富的多模态信息未被有效利用
3. **表征空间割裂**：多模态 embedding 作为固定特征平铺入模，与推荐任务目标存在语义鸿沟，且不可学习

本项目构建了一套从多模态表征训练 → 语义 ID 量化 → 精排模型应用的完整链路，将商品从无语义的 Hash ID 升级为兼具内容语义与协同信号的层次化离散表征。

---

## 一、多模态表征训练

### 1.1 为什么不用 CLIP，而用 MLLM

传统方案（如美团内部早期实践）使用 CLIP 提取图文特征，存在两个局限：
- 图像编码器和文本编码器的表征空间可能未对齐，需要额外融合
- CLIP 在通用数据上训练，对美团本地生活场景（外卖菜品、到店团购、酒旅等）的细粒度语义理解不足

本项目选用 **Qwen3-VL-2B**（多模态大语言模型，2B 参数）作为基础模型。相比 CLIP，MLLM 具备更深层的视觉-语言联合理解能力，能够从"商品视角"理解图文语义，而不只是做通用的图文匹配。

### 1.2 数据构建

针对美团各业务线（外卖、到餐、闪购、酒旅、医药等）**均匀采集** 1.7 亿图文样本，确保模型服务于全业务线而非单一场景。

每条样本的构成：
- **图片**：选取该商品曝光次数最多的图片（质量最优）
- **文本**：按 `{title} {cate_name} {brand_name} {description}` 顺序拼接，覆盖商品核心语义字段

同时对图片损坏、字段缺失的样本做过滤，保证训练数据质量。

### 1.3 训练方法

#### 图文解耦对比学习

采用**图文解耦输入策略**：图片和文本分别独立输入 MLLM，各自经过 Mean Pooling 得到表征向量，在同一语义空间内做对比学习。

以同一商品的图文表征互为正样本，Batch 内其他商品为负样本，使用 InfoNCE 损失：

$$L_i = -\log \frac{\exp(\text{sim}(I_i, T_i) / \tau)}{\exp(\text{sim}(I_i, T_i) / \tau) + \sum_{j \neq i} \exp(\text{sim}(I_i, T_j) / \tau)}$$

**为什么不用 MLLM 的多模态联合输入（图文拼接）？**
联合输入虽然能利用跨模态注意力，但会使图像和文本的表征耦合，无法分别提取独立的模态向量，不利于后续的图文 Concat 融合以及模态灵活组合。

#### 分布式负样本扩展

对比学习的效果高度依赖负样本数量，但直接增大 batch size 会导致显存爆炸。

解决方案：在分布式训练中，**跨 GPU 共享负样本**——将所有 P 个 GPU 上的 item 均纳入负样本池，负样本数量从 `B-1` 扩大到 `B×P-1`，在不增加单卡显存的前提下大幅提升训练效果。

#### MRL 多粒度表征学习

不同下游业务的计算资源差异很大，如果只输出固定维度（如 2048 维），计算资源受限的业务只能截断降维，导致信息损失；重新训练小模型则成本极高。

引入 **Matryoshka Representation Learning（MRL，套娃表征学习）**：在同一次训练中，对多个维度档位（32、64、128、256、2048）分别计算 InfoNCE loss 并加权求和。

这样训练出的模型，**前 N 维本身就是一个完整的、质量较高的小维度向量**，下游业务可根据计算资源自适应选择维度，无需重新训练。

MRL 的额外好处：多粒度 loss 起到正则化作用，强迫模型将最重要的语义信息压缩到向量前端，使低维表征更鲁棒。

训练结束后，将图像 Embedding 和文本 Embedding **Concat** 拼接，构建商品的完整多模态表征，融合两种模态的语义信息。

### 1.4 实验结果

在 10,000 条样本的测试集上，评估跨模态检索能力（图搜文 + 文搜图的 Recall 均值）：

| 模型 | Recall@1 | Recall@5 | Recall@10 |
|------|----------|----------|-----------|
| Baseline（未微调 Qwen3-VL-2B，2048维） | 0.0121 | 0.0318 | 0.0471 |
| 微调后全维度（2048） | **0.3678** | **0.5894** | **0.6583** |
| MRL 截断 64 维 | 0.3615 | 0.5853 | 0.6555 |
| MRL 截断 32 维 | 0.3524 | 0.5785 | 0.6500 |
| MRL 截断 16 维 | 0.3344 | 0.5630 | 0.6336 |

**关键结论**：
- 美团数据微调后，Recall@1 从 0.012 提升至 0.368，**提升约 30 倍**，验证了领域微调的必要性
- MRL 方法下，16 维表征相比全维度 Recall@1 仅损失 0.033，**以极低精度损失实现了多粒度自适应**，满足下游不同资源约束的需求

## 二、语义 ID 量化

### 2.1 为什么需要语义 ID，而不直接用连续 Embedding

多模态表征是高维连续向量，直接用于推荐模型存在两个问题：

1. **不可学习**：固定 Embedding 作为特征输入，无法通过推荐任务的梯度反向优化，与推荐目标存在语义鸿沟
2. **存储与检索开销大**：亿级商品的高维向量存储和 ANN 检索成本高，不适配高并发低延迟场景

语义 ID 通过量化将连续向量压缩为**层次化离散码字序列**（如 `330-641-530`），具备以下优势：
- 像普通 ID 特征一样做 Embedding Lookup，支持端到端联合训练
- 分层结构天然形成语义树，前缀相同的商品语义相近，支持高效的前缀索引召回
- 大幅压缩存储和检索开销

### 2.2 量化方法选择：为什么用 RQ-VAE 而非纯 RQ-Kmeans

业界常用的量化方法对比：

| 方法 | 优点 | 缺点 |
|------|------|------|
| 纯 Kmeans | 简单稳定 | 只有一层，信息损失大，无法生成层次化 ID |
| RQ-Kmeans | 多层残差，稳定易调参 | 无 Encoder-Decoder，无法端到端优化码本，码本质量上限低 |
| **RQ-VAE** | 含可学习 Encoder-Decoder，码本可联合优化，支持多维度 loss 约束 | 训练复杂，需要平衡多个 loss |

选择 **RQ-VAE** 的核心原因：RQ-Kmeans 的码本是静态的，无法通过下游任务的梯度反向优化；RQ-VAE 通过可学习的 Encoder-Decoder，使码本能够与推荐任务联合优化，且支持引入多维度损失约束来解决码本质量问题。

### 2.3 RQ-VAE 的核心问题与解决方案

纯 MSE 重建损失的 RQ-VAE 存在三个已知问题，本项目针对性地引入了对应解决方案：

#### 问题一：码本坍塌（Code Collapse）

**现象**：大量商品向量被映射到极少数码字，大部分码字从不被激活（"死码"），码本利用率极低。

**原因**：训练初期某些码字偶然被频繁选中，梯度更新使其越来越"好用"，形成马太效应，其他码字逐渐失活。

**解决方案：三重均衡机制**

1. **Balanced-Kmeans 初始化**：用均衡聚类初始化码本，保证每个码字在训练开始时就覆盖大致均等的数据区域，为后续训练提供良好起点。这是基础，如果初始化就严重不均衡，后续 loss 很难"救活"死码。

2. **Sinkhorn 均衡分配**：在每次量化时，不直接取最近邻码字，而是通过 Sinkhorn 算法求解最优传输问题，得到 batch 内均衡的软分配矩阵，再取 argmax。相当于在"选最近码字"的同时加了全局均衡约束。

3. **Diversity Loss**：对码本码字做聚类，训练时让量化结果与同簇其他码字也保持接近，鼓励同簇码字一起被激活，从训练目标层面防止死码。

三者的关系是递进而非替代：
```
Balanced-Kmeans（初始化均衡）
    → Sinkhorn（每步分配均衡）
        → Diversity Loss（训练目标均衡）
```

#### 问题二：层级冗余（Hourglass 现象）

**现象**：深层码本未能有效捕捉细节语义，码本利用率随量化层数增加而锐减，多层量化的信息增益递减。

**解决方案：残差正交化解耦损失 $\mathcal{L}_{ortho}$**

强制不同层级的量化向量在空间上尽量正交：

$$\mathcal{L}_{ortho} = \sum_{l=1}^{D-1} \sum_{p=l+1}^{D} \frac{|\langle e_l(k_l), e_p(k_p) \rangle|}{\|e_l(k_l)\| \|e_p(k_p)\|}$$

物理意义：第 1 层学"大类语义"，第 2 层学"中类语义"，第 3 层学"细粒度语义"，层间信息不重叠，每层码本都有独立的信息增益。

#### 问题三：协同信号缺失

**现象**：纯多模态量化只考虑内容语义，忽略用户行为信号。多模态语义相近 ≠ 用户行为相近。

**典型 case**：两家都卖"麻辣烫"的外卖店，图文极度相似，会被量化到同一码字；但一家在北京、一家在上海，用户行为上完全没有共现关系，被分在同一码字对推荐无意义。

**解决方案：量化阶段注入协同 InfoNCE**

> 注意：这里是在**量化阶段**加协同信号，而非表征阶段。两者目的不同：
> - 表征阶段加协同：让连续向量本身对齐用户偏好
> - **量化阶段加协同：让码字的分组边界符合用户行为逻辑**（哪些商品被分到同一码字）

做法：离线从 U2I/GR 召回模型导出全量商品的协同向量 $c_i$（携带用户行为信息）。在量化训练时，对每个商品同时有多模态表征向量 $h_i$ 和协同向量 $c_i$。

量化器对 $h_i$ 做残差量化，得到重构向量 $\hat{h}_i$（各层码字的累加），然后在 $\hat{h}_i$ 和 $c_i$ 之间做 InfoNCE：

$$\mathcal{L}_{cl} = -\sum_{i \in \text{Batch}} \log \frac{\exp(\text{sim}(\hat{h}_i, c_i) / \tau)}{\sum_{j \in \text{Batch}} \exp(\text{sim}(\hat{h}_i, c_j) / \tau)}$$

**为什么作用在 $\hat{h}_i$（重构向量）而非 $h_i$（原始表征）**：作用在重构向量上，意味着约束的是码字组合的语义——量化后的离散表示，在协同信号空间里也要对齐。直接影响码字边界的划分，使相同前缀的商品不只是图文相似，还是用户行为上相关的商品。

最终总损失：

$$\mathcal{L}_{total} = \mathcal{L}_{rec} + \lambda_1 \mathcal{L}_{quant} + \lambda_2 \mathcal{L}_{ortho} + \lambda_3 \mathcal{L}_{entropy} + \lambda_4 \mathcal{L}_{cl}$$

### 2.4 碰撞问题的工程兜底

即使有以上均衡机制，只要是聚类量化，就存在两个不同商品落到同一码字序列的概率（碰撞）。碰撞会导致下游无法从 SID 唯一还原到具体商品。

**解决方案：最后一层按序编码**

前 L-1 层做语义聚类，最后一层不做聚类，直接用商品在该簇内的顺序编号作为码字。这样在保留前缀语义结构的同时，**从工程上保证了 SID 与商品 ID 的一一映射**。

### 2.5 SID 质量评估指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 码本利用率 | 被激活的码字数 / 码本总大小 | 越高越好（>90%） |
| 碰撞率 | 重复码字序列数 / 总商品数 | 越低越好 |
| 码字分布熵 | 码字使用频率的信息熵 | 越高越好（分布越均匀） |
| 重构损失 | 重构向量与原始向量的 MSE | 越低越好 |
| 离线 Recall@K | 基于 SID 前缀的召回命中率 | 越高越好 |

## 三、精排模型应用：N-gram 多粒度 SID 入模

### 3.1 为什么选精排而非召回作为首选下游

召回侧（生成式召回 GR）需要 Encoder-Decoder 架构、Beam Search 在线推理、Torch 框架迁移等大量工程改造，周期长、风险高。

精排侧将 SID 作为 sideinfo 特征入模，**不改变召回链路，只新增特征**，工程风险最低，且美团内部已有多个场景验证了显著正向效果（美食频道 RPM +1.47%，神抢手 CTR +1.52%）。

### 3.2 N-gram 多粒度组合

假设商品有 3 层 SID：$(c_1, c_2, c_3)$，直接用 3 个独立特征会丢失层间的组合语义。

采用 **N-gram 多粒度组合**，生成 6 种子词组合：

$$\{c_1,\ c_2,\ c_3,\ c_1\text{\_}c_2,\ c_2\text{\_}c_3,\ c_1\text{\_}c_2\text{\_}c_3\}$$

每种组合代表一种粒度的语义：
- $c_1$：最粗粒度，代表大类（如"正餐"）
- $c_1\text{\_}c_2$：中粒度（如"正餐-火锅"）
- $c_1\text{\_}c_2\text{\_}c_3$：最细粒度（如"正餐-火锅-川式火锅"）

各组合使用**独立的 Embedding 表**（不共享参数），每种粒度分配 32 维 Embedding，最终得到 $32 \times 6 = 192$ 维的 SID 表征。

**为什么不共享 Embedding 表**：不同粒度的语义空间不同，共享参数会导致粗粒度和细粒度的语义混淆，独立 Embedding 表让模型自由学习每个粒度的语义。

### 3.3 与用户行为序列的交叉

将 SID 特征拼接到用户行为序列的 sideinfo 中（每个历史交互商品附带其 SID），通过 **target-attention** 机制与目标商品进行交叉：

```
用户行为序列：[item_1(sid_1), item_2(sid_2), ..., item_n(sid_n)]
                                    ↓ target-attention
                              目标商品 SID
                                    ↓
                            用户对当前商品的语义兴趣表达
```

这样模型不只是在 item ID 空间里做兴趣匹配，还能在语义空间里捕捉"用户喜欢什么类型的商品"，对长尾商品和新品效果提升尤为明显（因为新品没有历史交互数据，但 SID 携带了内容语义，能与用户历史的语义偏好对齐）。

### 3.4 与原始 Embedding 的融合

SID 表征（可学习，192维）与原始多模态 Embedding（固定，经全连接降维）做 **Sum Pooling** 融合：

```
最终表征 = SID表征（可学习，语义对齐）
         + 原始多模态Embedding（固定，语义基础）
```

保留原始 Embedding 的原因：SID 是离散化后的有损压缩，原始 Embedding 提供更完整的连续语义信息作为补充，两者互补。实验表明，单独用 SID 或单独用原始 Embedding 效果均不如融合（参考神抢手实验：SID+原始Embedding > 纯SID > 纯原始Embedding）。

## 四、多模态表征的下游应用方向

多模态表征生产出来后，可以在推荐系统的不同层级发挥作用。以下按应用层次梳理。

### 4.1 精排层：序列建模增强（美团 MTMR 实践）

这是美团猜喜项目的核心应用方式，分两个阶段：

- **V-GSU（粗筛）**：原始 GSU 只用 ID 特征从终身行为序列中检索候选，ID 稀疏时效果差。引入 vlm_emb 做视觉相似检索，用图片先验弥补 ID 稀疏的缺陷，更准确地从长序列中捞出有效行为。
- **V-ESU（精细建模）**：在序列每个 item 的 sideinfo 中拼入 `W_proj × vlm_emb`，与 ID embedding、静态属性、时空特征一起参与 target-attention 计算。多模态信息在 ID embedding 学习不足时发挥兜底作用，保证模型有足够的语义表达能力。

同时，在模型内部通过 AlignLoss 对 vlm_emb 与 itemID embedding 做**端到端对齐**，联合主任务 loss 反向传播，使多模态语义空间持续向推荐协同信号靠拢。

### 4.2 召回层：多模态向量召回与难负样本挖掘

- **ANN 向量召回**：将 vlm_emb 作为 item 的向量表示建 ANN 索引，在召回阶段做视觉相似召回（以图搜图）。对新 item 天然友好，不依赖协同信号，是冷启动场景的重要补充。
- **难负样本挖掘**：训练双塔召回模型时，用 vlm_emb 找视觉相似但用户未点击的 item 作为难负样本，让模型学会区分视觉相似但用户偏好不同的供给，提升召回模型的细粒度判别能力。

### 4.3 冷启动：新 item 的 ID Embedding 初始化

新 item 上线时 ID embedding 随机初始化，收敛慢、效果差。可将 vlm_emb 经投影后作为 ID embedding 的初始值，让新 item 从训练开始就具备合理的语义表示，加速收敛。本质是把视觉先验迁移到协同空间，字节、快手均有类似实践。

### 4.4 用户侧：多模态 User 兴趣表征

不只对 item 建多模态向量，也可对用户历史行为序列做聚合，得到 user 的多模态兴趣表征：

```
user_vlm = AttentionPooling([vlm_emb(item) for item in user_history])
```

用 user_vlm 与 target item 的 vlm_emb 做匹配，捕捉用户的视觉层面兴趣偏好。阿里 MAKE、小红书 NoteLLM-2 均有类似思路。

### 4.5 跨模态桥接：文字 Query 检索图片 Item

用多模态模型将图片和文字映射到同一语义空间，实现文字 query 检索图片 item，解决搜索/推荐中图文异构问题。在搜索场景更为常见，推荐场景中可用于用户文字标签与商品图片的语义匹配。

### 4.6 离散化路线：量化编码做序列建模

快手 QARM 的思路：用残差量化（RVQ）将连续 vlm_emb 编码为离散 token 序列，复用 NLP 的 Transformer 架构对这些 token 做序列建模。代价是量化存在信息损失，完全舍弃了原始连续表征。本项目的 SID 量化方案也属于这一路线，但通过协同 InfoNCE 和正交化损失保留了更多有效信息。

### 4.7 小结：应用层次全景

```
多模态表征的应用层次：

召回层  → ANN 向量召回、难负样本挖掘
精排层  → 序列建模（V-GSU / V-ESU）、特征拼接、端到端对齐
冷启动  → ID Embedding 初始化
用户侧  → User 多模态兴趣表征
跨模态  → 图文统一语义空间
离散化  → 量化编码 + 序列建模（SID 路线）
```

## 六、简历描述（可直接使用）

**美团 | 猜你喜欢推荐 | 多模态表征与语义ID体系建设**

- **多模态表征训练**：基于 Qwen3-VL-2B，在美团全业务线 1.7 亿图文样本上进行对比学习微调，采用图文解耦输入策略，引入跨 GPU 负样本扩展（负样本规模从 B-1 扩大至 B×P-1）及 MRL 多粒度表征学习（16～2048 维自适应输出），跨模态检索 Recall@1 从 0.012 提升至 0.368（**提升约 30 倍**），16 维截断相比全维度 Recall@1 仅损失 0.033，验证了低资源场景下的可用性

- **语义 ID 量化**：基于上述多模态表征，采用 RQ-VAE 残差量化构建层次化语义 ID 体系，针对码本坍塌问题引入 Balanced-Kmeans 初始化 + Sinkhorn 均衡分配 + Diversity Loss 三重均衡机制，针对层级冗余问题引入残差正交化解耦损失（$\mathcal{L}_{ortho}$），同时在量化阶段通过 InfoNCE 对比学习注入协同信号（重构向量与 U2I 协同向量对齐），使码字分组同时兼顾内容语义与用户行为相似性，最后一层按序编码保证商品与 SID 的一一映射

- **精排模型应用**：将 3 层 SID 以 N-gram 多粒度组合方式（6 种子词 × 独立 Embedding 表 × 32 维 = 192 维）作为 sideinfo 引入精排模型，与用户行为序列进行 target-attention 交叉，并与原始多模态 Embedding 做 Sum Pooling 融合，实现端到端联合训练；对标美团内部同类方案，精排模型离线 GAUC 提升显著，线上实验 RPM/CTR 正向（待补充线上数据）

**美团 | 猜你喜欢推荐 | UniRank 统一精排特征建模框架**

> 详细技术设计见：`unirank_project.md`

- **AutoToken 自动特征分组**：针对 400+ 异构特征人工分组维护成本高、跨场景复用差的问题，设计可学习选择矩阵 $S \in \mathbb{R}^{n_g \times n_f}$，通过 Top-k + Softmax 加权聚合实现端到端可微分的自动特征分组，消除人工特征工程；消融实验表明 AutoToken 效果媲美领域专家人工分组，CTR-GAUC 相比随机分组提升 **+8bp**

- **序列与特征交叉统一建模**：针对序列建模与特征交叉串行割裂、行为细节不可逆损失的问题，将行为序列 S-token 与非序列特征 NS-token 拼接成统一序列，通过 Mixed Causal Attention 使非序列特征 token 直接 attend 每个行为 token，同时完成序列聚合与特征交叉；相比"先序列后交叉"的串行方案，CTR-GAUC 额外提升 **+13bp**

- **共享+场景稀疏 MoE 多场景建模**：针对多场景模型碎片化、跨场景知识无法共享的问题，设计 Shared Dense Expert（全场景通用知识）+ Scenario Sparse Expert（场景特有知识，手动 bonus 保证当前场景专家必激活）的混合 MoE 架构；引入 MLoRA 低秩适配器支持新场景快速接入（参数量仅为全量微调的 1%）；整体方案 CTR-GAUC 提升 **+27bp**，CTCVR-GAUC 提升 **+20bp**
