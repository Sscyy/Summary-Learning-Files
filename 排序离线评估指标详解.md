# 搜索排序离线评估指标详解

> 基于《服务零售精排引入大模型语义表征二期》Launch Review 文档的学习整理

---

## 一、AUC 的本质

AUC（Area Under Curve）最直观的理解是一个**概率**：

> 随机取一个正样本和一个负样本，模型给正样本打的分高于负样本的概率。

- AUC = 0.5：等于瞎猜（随机水平）
- AUC = 1.0：完美排序
- AUC = 0.9：随机抽一个点击商品和一个未点击商品，有 90% 的概率模型给点击的那个打更高的分

---

## 二、Q2I-Click-AUC 的具体计算过程

### 第一步：构造测试集

从精排日志里取 (Query, Item, label) 三元组：
- 用户点击了 → 正样本，label = 1
- 曝光但未点击 → 负样本，label = 0

### 第二步：用表征算相似度分

把 Query 和 Item 分别用大模型编码成向量，计算余弦相似度作为打分：

$$\text{sim}(q, i) = \frac{q \cdot i}{|q| \cdot |i|}$$

### 第三步：遍历所有正负样本对计算 AUC

对每一对（正样本 $p$，负样本 $n$）：
- $\text{sim}(q, p) > \text{sim}(q, n)$ → 计 1 分
- $\text{sim}(q, p) = \text{sim}(q, n)$ → 计 0.5 分
- $\text{sim}(q, p) < \text{sim}(q, n)$ → 计 0 分

$$\text{AUC} = \frac{\text{总得分}}{\text{正负样本对总数}}$$

### 具体例子

搜索词"火锅"，3 个正样本（点击）、3 个负样本（未点击）：

| 样本 | 标签 | 相似度 |
|------|------|--------|
| 海底捞 | 正 | 0.85 |
| 小龙坎 | 正 | 0.72 |
| 呷哺呷哺 | 正 | 0.61 |
| 麦当劳 | 负 | 0.30 |
| 星巴克 | 负 | 0.25 |
| 药店 | 负 | 0.10 |

正负样本对共 3×3 = 9 对，全部正样本分数高于负样本，AUC = 9/9 = **1.0**。

## 三、各 AUC 指标的区别

| 指标 | 正样本定义 | 打分方式 | 衡量的是 |
|------|-----------|---------|---------|
| Q2I-Click-AUC | 用户点击的 (Query, Item) 对 | Query 和 Item 表征的余弦相似度 | 表征对点击行为的语义匹配质量 |
| Q2I-Order-AUC | 用户下单的 (Query, Item) 对 | 同上 | 表征对转化行为的语义匹配质量 |
| Q2I-Corr-AUC | 人工标注为相关的 (Query, Item) 对 | 同上 | 表征与人类语义判断的一致性，不受行为噪声干扰 |
| I2I-AUC | 同一 Query 下被点击的两个 Item 互为正样本 | 两个 Item 表征的余弦相似度 | 商品表征空间本身的质量 |
| Q2D-Corr-AUC | 人工标注为相关的 (Query, Deal) 对 | Query 和下挂商品表征的余弦相似度 | 搜索词与下挂商品的语义匹配质量（二期新增） |

**为什么 Order-AUC 绝对值通常高于 Click-AUC？**

下单信号比点击更纯净（用户下单说明强烈满意），正负样本区分更容易，所以 AUC 绝对值更高。但 Click-AUC 的提升空间更大，因为点击信号更嘈杂、基数更低。

## 四、NDCG 指标

### 核心思想

NDCG（Normalized Discounted Cumulative Gain）不仅关注排序对不对，还关注**排在第几位**——越靠前的位置权重越高。

### 计算公式

$$\text{NDCG} = \frac{\text{DCG}}{\text{IDCG}}$$

其中 DCG（实际排序得分）：

$$\text{DCG} = \sum_{i=1}^{n} \frac{\text{label}_i}{\log_2(i+1)}$$

IDCG 是理想排序（所有正样本排在最前面）时的 DCG，用于归一化。

位置权重示例：
- 第 1 位：$1 / \log_2(2) = 1.0$
- 第 2 位：$1 / \log_2(3) \approx 0.63$
- 第 5 位：$1 / \log_2(6) \approx 0.39$
- 第 10 位：$1 / \log_2(11) \approx 0.29$

### Q2I-Click-NDCG 的计算方式

对每一个 Query，把它对应的所有候选 Item 按表征相似度从高到低排列，计算该排列的 NDCG，最后对所有 Query 取平均。

## 五、AUC 和 NDCG 能用同一个测试集吗？

**可以，而且通常就是同一份数据。** 同一批 (Query, Item, label) 三元组，分别跑两种计算逻辑，得到两个不同维度的指标。

### 两个指标的侧重点不同

| | AUC | NDCG |
|--|-----|------|
| 关注范围 | 全局所有正负样本对 | 每个 Query 内部的排序 |
| 位置敏感性 | 不敏感，只看相对顺序 | 敏感，越靠前权重越高 |
| 适用场景 | 整体区分能力评估 | 搜索/推荐头部结果质量评估 |

### 两个指标会出现不一致

文档中 LoRA 微调 vs 全参数微调的对比就是典型案例：

| 微调方式 | Q2I-Click-AUC | Q2I-Click-NDCG |
|---------|--------------|----------------|
| 全参数微调 | **0.6543** | 0.7464 |
| LoRA 微调 | 0.6444 | **0.7475** |

LoRA 的 AUC 更低，但 NDCG 更高。最终选择 LoRA，因为搜索场景用户主要看前几个结果，**NDCG 对业务的指导意义更直接**。

## 六、离线测试集的构造原则

### 基本要求

- 与训练集**严格按时间切分**，防止数据泄露
- 构造好后**固定不动**，所有模型变体在同一测试集上对比，结果才有可比性
- 是**领域专用**的，不是通用测试集——服务零售搜索的测试集不能直接用于其他场景

### 负样本质量是关键

测试集里的负样本通常是"曝光未点击"，但这些样本本身是被当前排序系统选出来曝光的，天然偏向相关性较高的商品，**真正的难负样本严重不足**，可能导致离线 AUC 虚高。

检验方法：把测试集里的负样本换成随机采样的商品，看 AUC 是否大幅下降。如果下降很多，说明原来的 AUC 是在"简单"测试集上刷出来的。

## 七、离线 AUC 涨了，线上效果不好怎么排查？

### 排查优先级（从高到低）

**1. 确认线上效果"不好"的具体表现**

先搞清楚是什么样的"不好"，因为不同表现指向不同根因：
- 所有场景都不好 vs 某些子场景负向
- 核心指标没变化 vs 明显负向
- 刚上线就不好 vs 跑了几天才变差

**2. 验证模型是否真的生效**

- **表征覆盖率**：大量请求拿不到表征向量（新商家、长尾 Query），会退化到零向量，等于没有语义信息。文档中 Query 曝光覆盖率仅 79.59%，有 20% 的请求实际没有享受到语义表征收益。
- **特征接入正确性**：线上读到的表征向量和离线训练时用的是否一致？版本对不上、KV 读取超时降级都会导致实际用的表征和预期不符。
- **特征是否被模型有效利用**：可以做"强制置零"实验——把表征特征全部置零，如果线上指标没有任何变化，说明这个特征根本没有被模型有效利用。

**3. 排查离线和线上的分布差异**

- **时间窗口**：测试集时间和上线时间差太远，流量分布已经变化，离线 AUC 参考价值下降。
- **负样本质量**：见上文，离线负样本偏简单，线上遇到真实难样本就撑不住。
- **评估目标对齐**：离线用 Q2I-Click-AUC 评估，线上优化 CTR/CTCVR，中间还经过精排融合、混排校准等多个环节，每个环节都可能引入偏差。

**4. 排查系统联动问题**

- **校准模块是否适配新模型**：精排引入新特征后，输出分值分布会变化。如果下游校准模块是基于旧模型拟合的，新模型上线后校准会失效。文档里就是这个问题——实验组预估更准了，但混排校准没有同步更新，反而把准确的预估校准歪了，导致易生活和医药健康曝光被压低。
- **混排策略是否依赖精排绝对分值**：如果混排按各业务绝对分值竞争曝光，精排分值分布的任何变化都会影响各业务曝光占比。

**5. 细粒度线上下钻**

- 按**表征覆盖情况**分桶：有表征 vs 无表征的请求，效果是否有差异
- 按**相似度分桶**分析：高相似度桶的 CTR 是否符合预期，如果高相似度桶 CTR 反而低，说明表征语义对齐方向有问题
- 按 **Query 类型**分析：精确匹配 Query（搜索词=商家名）vs 语义匹配 Query（搜索品类/意图），语义表征对后者帮助更大

---

## 八、离线指标和线上指标的关系

```
离线 AUC/NDCG 提升
        ↓
    筛选门槛（不涨则直接排除方案）
        ↓
    值得上线做 A/B 实验
        ↓
    线上实验结果才是最终裁判
```

**离线指标是筛选门槛，不是最终判断。** 离线涨了说明"值得试试"，不代表线上一定正向。离线没涨则基本可以直接排除，不用浪费线上实验资源。
