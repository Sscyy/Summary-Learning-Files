# NCE Loss 及对比学习损失函数完整知识体系

---

## 第一章：为什么需要这些Loss？——问题的起源

### 1.1 语言模型的基本任务

语言模型最基础的任务是：**给定上下文，预测下一个词是什么。**

比如句子"我喜欢吃___"，模型要从词表里找出最合适的词填进去。

### 1.2 传统Softmax的致命问题

最自然的做法是用Softmax：对词表里每个词打分，然后归一化成概率。

$$P(w | \text{context}) = \frac{e^{s(w)}}{\sum_{w' \in V} e^{s(w')}}$$

**问题在于分母**：要对词表里**全部V个词**求和。

```
词表大小 V = 10万
每次预测：计算10万次exp + 求和 + 归一化
每次反向传播：更新10万个词的参数
训练一个epoch：数亿次这样的操作

→ 慢到无法接受
```

这就是所有采样类Loss的起点：**如何在不算全部词的情况下，让模型学到正确的东西？**

## 第二章：NCE Loss——把难题变简单

### 2.1 核心思路：偷换问题

NCE（Noise Contrastive Estimation，噪声对比估计）的思路是：

> **不再问"这10万个词里哪个是正确的"（多分类），而是问"这个词是真货还是冒牌货"（二分类）。**

### 2.2 正负样本如何构造

```
从语料里取出：(上下文="我喜欢吃", 真实词="苹果")  → 正样本，label=1

从噪声分布q(w)里随机采样K个词：
[桌子, 跑步, 天空, 电脑, 香蕉]  → 负样本，label=0
```

**噪声分布 $q(w)$ 通常用词频分布**，实践中常用 $q(w) \propto \text{词频}^{0.75}$，适当压低超高频词的权重。

**一条训练样本 = 1个正样本 + K个负样本**，负样本每次动态采样，不固定。

### 2.3 打分与修正

模型对每个候选词打原始分数 $s(w)$（神经网络输出的logit），然后做修正：

$$\text{修正后分数} = s(w) - \log(k \cdot q(w))$$

**为什么要修正？**

不修正的问题：高频词（如"的"）会被频繁采为负样本，模型为了把它判成冒牌货会拼命压低它的分数，导致 $s(w)$ 被词频信息污染，高频词分数系统性偏低。

修正的作用：把噪声分布的先验影响从模型分数里剔除，让 $s(w)$ 只反映纯粹的语义相关性。

**理论保证**：
| 版本 | 收敛结果 |
|---|---|
| 不做修正（负采样） | $s(w)$ 是被词频污染的扭曲分布 |
| 做修正（NCE） | $s(w)$ 收敛到真实数据分布 $\log P_{data}(w)$ |

### 2.4 NCE Loss 公式

$$P(\text{真货} | w) = \sigma\bigl(s(w) - \log(k \cdot q(w))\bigr)$$

$$\mathcal{L}_{NCE} = -\left[\log P(\text{真货}|w_{pos}) + \sum_{i=1}^{k}\log\bigl(1 - P(\text{真货}|w_{neg}^i)\bigr)\right]$$

读法：
- 第一项：正样本被判为真货的概率，要最大化（越接近1越好）
- 第二项：每个负样本被判为冒牌货的概率，也要最大化（越接近0越好）
- 加负号：变成最小化Loss，符合梯度下降习惯

### 2.5 完整数据流转

```
第①步：上下文词 → Embedding层 → 上下文向量 h
第②步：对(1+K)个候选词分别查Embedding → 候选词向量 e(w)
第③步：打分  s(w) = h · e(w)
第④步：修正  score(w) = s(w) - log(k · q(w))
第⑤步：sigmoid → P(真货|w)
第⑥步：代入NCE Loss公式 → 得到标量Loss
第⑦步：反向传播，更新参数
```

### 2.6 反向传播的直觉

**对正样本的梯度：**
$$\frac{\partial \mathcal{L}}{\partial e(w_{pos})} = -(1 - P(\text{真货}|w_{pos})) \cdot h$$

- $P$ 离1越远，梯度越大，更新越猛
- 效果：把正样本词向量往上下文向量方向拉近

**对负样本的梯度：**
$$\frac{\partial \mathcal{L}}{\partial e(w_{neg}^i)} = P(\text{真货}|w_{neg}^i) \cdot h$$

- $P$ 离0越远，梯度越大，更新越猛
- 效果：把负样本词向量从上下文向量方向推远

### 2.7 计算量对比

```
原始Softmax：每次更新需要计算 O(V) = O(10万)
NCE Loss：每次更新只需要计算 O(K+1) = O(6)

提速：数百倍到数千倍
```

## 第三章：负采样（Negative Sampling）——NCE的工程简化版——NEG loss

### 3.1 什么是负采样

负采样是Word2Vec实际使用的方法，可以理解为**去掉修正项的NCE**：

```
直接：s(w) → sigmoid → P(真货|w) → 二元交叉熵Loss
```

不做 $\log(k \cdot q(w))$ 的修正。

### 3.2 负采样 vs NCE

| | 负采样 | NCE |
|---|---|---|
| 修正项 | 无 | 有 |
| 理论严谨性 | 学到的分布被词频污染 | 收敛到真实数据分布 |
| 工程实现 | 更简单 | 稍复杂 |
| 实践效果 | 词向量质量很好 | 理论上更优 |

> 实践中负采样效果已经很好，Word2Vec默认使用负采样而非严格的NCE。

## 第四章：Sampled Softmax——换一种缩减方式

### 4.1 核心思路：缩小候选池，而非换题型

NCE是把多分类变成二分类。Sampled Softmax的思路不同：

> **问题性质不变，还是多分类；但不和全部10万个词比，只和采样出来的一小撮词比。**

### 4.2 公式

$$\mathcal{L}_{sampled\_softmax} = -\log \frac{e^{s(w_{pos})}}{e^{s(w_{pos})} + \sum_{i=1}^{K} e^{s(w_{neg}^i)}}$$

分母从对全部V个词求和，缩减为只对 **(1+K)** 个词求和。

### 4.3 同样需要修正

原因和NCE一样，修正方式：

$$\hat{s}(w) = s(w) - \log q(w)$$

代入公式后，分母里每个词的分数都经过词频校正。

### 4.4 Sampled Softmax vs NCE 本质区别

用比喻说清楚：

```
同样给你6张卡片：[苹果✓] [桌子✗] [跑步✗] [天空✗] [电脑✗] [香蕉✗]

NCE问的是：
"每张卡片，你觉得它是真货吗？"  ← 6个独立的是/否判断
用sigmoid，每张卡片独立计算，互不影响

Sampled Softmax问的是：
"这6张里，哪一张是真货？"  ← 6选1的竞争
用softmax，6张卡片互相竞争，概率之和=1
```

**梯度的差异：**

| | NCE | Sampled Softmax |
|---|---|---|
| 梯度耦合性 | 每个词梯度独立 | 所有候选词梯度耦合 |
| 正样本梯度 | 只和自己的sigmoid输出有关 | 和所有候选词的softmax输出都有关 |
| 理论严谨性 | 收敛到真实分布（无偏） | 有偏估计，是完整Softmax的近似 |

### 4.5 理论严谨性

```
原始Softmax分母 = Σ(全部V个词的exp分数)  ← 精确
Sampled Softmax分母 = Σ(1+K个词的exp分数) ← 近似，有偏
```

有偏不代表没用——K足够大时效果很好，YouTube、Facebook等工业推荐系统广泛使用。

## 第五章：InfoNCE Loss——对比学习的灵魂

### 5.1 全新的背景：自监督表示学习

前面三种Loss的出发点都是**加速语言模型训练**，本质是工程提速工具。

InfoNCE出自2018年DeepMind的论文《Representation Learning with Contrastive Predictive Coding（CPC）》，要解决的是完全不同的问题：

> **如何在没有任何人工标签的情况下，让模型自己学到有意义的特征表示？**

这是**自监督对比学习**的核心命题。

### 5.2 理论基础：最大化互信息

**互信息**衡量的是两个变量之间共享了多少信息量。

InfoNCE的目标：

> 最大化"上下文表示"和"目标表示"之间的互信息
>
> 说人话：让模型学会，从一个样本的一个视角，能预测出它另一个视角的特征

互信息直接计算极其困难，InfoNCE提供了一个**互信息的可优化下界**——最小化InfoNCE Loss等价于最大化互信息。

### 5.3 正负样本的构造——最关键的创新

这是InfoNCE和前面所有方法最根本的不同：

**之前的方法：**
```
正样本：语料里天然存在的真实词对
负样本：从词频分布随机采样
```

**InfoNCE：**
```
正样本：需要精心构造！通过数据增强制造"同一事物的不同视角"

图片领域：同一张图 → 随机裁剪+旋转+变色 → 两个增强版本互为正样本
文本领域：同一句话 → 不同的dropout mask → 两个版本互为正样本
音频领域：同一段音频 → 不同时间窗口 → 两段互为正样本

负样本：batch内其他样本（不需要额外采样，直接用）
```

> **正样本的构造方式定义了模型学什么**——你告诉模型"这两个东西是相似的"，模型就会去学它们之间共同的本质特征。

### 5.4 InfoNCE Loss 公式

$$\mathcal{L}_{InfoNCE} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp(q_i \cdot k_i^+ / \tau)}{\sum_{j=1}^{N} \exp(q_i \cdot k_j / \tau)}$$

| 符号 | 含义 |
|---|---|
| $q_i$ | 查询向量（query），样本的一个视角编码 |
| $k_i^+$ | 正样本键向量（key），同一样本另一个视角的编码 |
| $k_j$ | 分母里所有候选（1个正样本 + N-1个负样本） |
| $\tau$ | 温度系数，控制分布尖锐程度 |

### 5.5 InfoNCE 和 Sampled Softmax 的公式关系

两者并排对比：

$$\text{Sampled Softmax: } -\log \frac{e^{s(w_{pos})}}{e^{s(w_{pos})} + \sum_{i=1}^{K} e^{s(w_{neg}^i)}}$$

$$\text{InfoNCE: } -\log \frac{\exp(q \cdot k^+ / \tau)}{\exp(q \cdot k^+ / \tau) + \sum_{j=1}^{K} \exp(q \cdot k_j^- / \tau)}$$

**计算结构完全一致**，区别在于：

| 维度 | Sampled Softmax | InfoNCE |
|---|---|---|
| 解决问题 | 大词表加速 | 自监督表示学习 |
| 正样本来源 | 语料天然存在 | 数据增强构造 |
| 负样本来源 | 词频分布采样 | batch内其他样本 |
| 温度系数 | 无（τ=1） | 有，且至关重要 |
| 理论意义 | Softmax的有偏近似 | 互信息的下界 |

> **InfoNCE是Sampled Softmax在自监督表示学习领域的"精神续作"——借用了计算外壳，但目标和灵魂完全不同。**

### 5.6 温度系数 τ 的作用

这是InfoNCE独有的关键参数：

```
τ 很小（如0.07）：
  分布很尖锐，模型对相似度差异极度敏感
  正样本必须和query非常近，负样本必须非常远
  → 特征区分度高，但训练容易不稳定

τ 很大（如1.0）：
  分布很平滑，模型对差异不敏感
  正负样本的分数差异被抹平
  → 训练稳定，但特征质量差，学不到有意义的表示
```

> **τ 的本质：控制模型对"相似度差异"的放大倍数。** 实践中τ通常取0.07~0.2，需要根据任务调整。

### 5.7 完整数据流转（以图片对比学习为例）

```
第①步：取一张图片，做两次不同的数据增强
        原图 → 增强A → query向量 q
        原图 → 增强B → 正样本key向量 k+

第②步：batch内其他图片的增强版本作为负样本
        图片C增强 → k1-
        图片D增强 → k2-
        ...

第③步：分别通过编码器（如ResNet）得到向量
        q, k+, k1-, k2-, ... 都是L2归一化后的向量

第④步：计算相似度（点积）
        q·k+ = 0.85（正样本，应该高）
        q·k1- = 0.12（负样本，应该低）
        ...

第⑤步：除以温度系数τ，代入InfoNCE公式
        L = -log(exp(0.85/τ) / (exp(0.85/τ) + exp(0.12/τ) + ...))

第⑥步：反向传播
        让 q 和 k+ 的向量在特征空间靠近
        让 q 和所有 k- 的向量在特征空间远离
```

## 第七章：一些重要的补充知识点

### 7.1 负样本数量K怎么选？

| K值 | 效果 |
|---|---|
| K太小（1~2） | 负样本太少，区分能力弱，学得不充分 |
| K=5~20 | NCE/负采样的甜点区（Word2Vec推荐） |
| K=数百~数千 | Sampled Softmax在推荐系统里的常用范围 |
| K→V（全量） | 退化为原始Softmax，失去加速意义 |

理论上K越大越接近真实分布，但收益递减，需要权衡计算成本。

### 7.2 对比学习中的"模式坍塌"问题

InfoNCE有一个潜在风险：如果负样本不够多或不够难，模型可能学到"捷径"——比如所有样本都映射到同一个向量，Loss也能很低。

这叫**模式坍塌（Mode Collapse）**。

解决方案：
- **SimCLR**：用超大batch（4096），保证足够多的负样本
- **MoCo**：用动量编码器 + 队列，维护大量负样本
- **BYOL/SimSiam**：完全不用负样本，用其他机制避免坍塌

### 7.3 InfoNCE的一个重要变体：监督对比学习

原始InfoNCE是自监督的（正样本靠数据增强构造）。

2020年的**Supervised Contrastive Learning**将其扩展到有标签场景：

```
同一类别的所有样本互为正样本
不同类别的样本互为负样本
```

这样每个样本可以有多个正样本，Loss变为：

$$\mathcal{L} = \sum_{i} \frac{-1}{|P(i)|} \sum_{p \in P(i)} \log \frac{\exp(q_i \cdot k_p / \tau)}{\sum_{j \neq i} \exp(q_i \cdot k_j / \tau)}$$

效果往往优于传统交叉熵Loss。
