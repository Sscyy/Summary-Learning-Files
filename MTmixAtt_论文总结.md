# MTmixAtt 论文总结

> **一句话概括**：通过自动特征分组（AutoToken）和混合专家架构（Shared Dense + Scenario Sparse MoE），实现无需人工干预的特征建模和跨场景迁移，在美团多场景推荐中超越 RankMixer 等基线。

---

## 一、背景：为什么要做这件事？

### 1.1 工业推荐的三大痛点
| 痛点 | 说明 |
|------|------|
| **人工特征工程** | 特征分组依赖人工设计，主观性强，跨场景难以复用 |
| **架构碎片化** | 不同特征类型（密集/稀疏/序列）需要不同网络结构（MLP/Transformer/GNN），难以统一 |
| **跨场景迁移差** | 为一个场景（如首页）优化的模型难以迁移到另一场景（如团购），需要昂贵的重新工程 |

### 1.2 与 RankMixer 的关系
- MTmixAtt **直接引用 RankMixer** 作为对比基线
- 声称在相同参数规模下，各项指标**超越 RankMixer**
- 但：数据集不同（美团 TRec vs 抖音）、任务不同（CTR/CTCVR vs Finish/Skip）、场景设置不同（多场景 vs 单场景）

### 1.3 核心目标
> **构建一个统一框架，同时解决自动特征分组、异构特征建模、跨场景适配三个问题**

---

## 二、核心思想：自动化优先的架构设计

### 2.1 与 RankMixer 的设计哲学对比
| 维度 | RankMixer | MTmixAtt |
|------|-----------|----------|
| **首要目标** | 硬件效率（高 MFU） | 自动化（消除人工干预） |
| **特征分组** | 人工语义分组 | **AutoToken 自动分组** |
| **Token 交互** | 固定重排（零参数） | **可学习混合矩阵** |
| **MoE 设计** | Per-token 独立专家 | **共享密集 + 场景稀疏混合** |
| **多场景** | 未涉及 | **核心能力，显式建模** |
| **训练策略** | DTSI（训练 Dense，推理 Sparse） | 始终 Dense（Sigmoid 激活） |

### 2.2 关键权衡
- **RankMixer**：牺牲一定表达能力换取极致硬件效率
- **MTmixAtt**：接受一定计算开销换取自动化和灵活性

## 三、方法详解：数据如何流转

### 3.1 整体架构（宏观视角）
```
输入层：400+ 异构特征（6 大类：序列/多模态/用户属性/物品属性/交叉特征/场景特征）
    ↓
AutoToken（自动特征分组）──► n_g 个 tokens
    ↓
MTmixAtt Block × N 层：
  ├─ Learnable Token Mixing（可学习混合矩阵）
  ├─ Shared Dense MoE（共享密集专家）
  └─ Scenario Sparse MoE（场景稀疏专家）─► 多场景适配
    ↓
场景特定的预测头（主场景: DNN / 其他场景: MLoRA 适配器）
    ↓
输出：CTR / CTCVR 预测
```

### 3.2 第一层：AutoToken（自动特征分组）

**为什么**：消除人工特征分组的主观性和不一致性

**怎么做**：
1. **特征对齐**：每个特征通过专属 DNN_i 投影到统一维度 d
2. **可学习选择矩阵**：S ∈ R^{n_g × n_f}，n_g 为分组数，n_f 为特征数
3. **Top-k 选择 + Softmax 加权**：对每个组，从 S 的行中选 Top-k 个特征，加权聚合

**关键公式**：
- 公式 4：X = f(φ_k(S_{j,:}), X̂) · ω_k(S_{j,:})
- 公式 5：ω_k(S_{j,:}) = softmax(top-k(S_{j,:}))

**关键洞察**：通过 softmax 使 TopK 操作可微分，实现端到端学习

**收益**：数据驱动的最优分组，跨场景自适应

**代价**：需要训练 S 矩阵，增加一定计算开销

### 3.3 第二层：Learnable Token Mixing（可学习混合矩阵）

**与 RankMixer 的核心区别**：
| 方面 | RankMixer | MTmixAtt |
|------|-----------|----------|
| **方式** | 固定重排（Split → Shuffle → Merge） | 可学习矩阵变换 |
| **参数** | 0 | h × n_g² |
| **复杂度** | O(n_g·d) | O(n_g²·d) |
| **GPU 效率** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **表达能力** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**数据流转**：
1. 转置：X ∈ R^{d×n_g} → X^T ∈ R^{n_g×d}
2. 分 heads：X^T = [X^(1) ‖ X^(2) ‖ ... ‖ X^(h)]
3. 可学习混合：Y^(i) = M_i^T · X^(i)，M_i ∈ R^{n_g×n_g}
4. 残差连接：Z^(i) = X^(i) + Y^(i)
5. 拼接 heads：Z = [Z^(1) ‖ ... ‖ Z^(h)]

**关键公式**：
- 公式 8：Y^(i) = M_i^T · X^(i)
- 公式 9：Z^(i) = X^(i) + M_i^T · X^(i)

**初始化重要性**：正交初始化（M_i ≈ I）效果最好，初始时近似恒等映射，逐步学习更复杂模式

### 3.4 第三层：Shared Dense MoE（共享密集专家）

**设计动机**：RankMixer 的 Per-token 专家完全独立，缺乏跨 token 知识共享

**数据流转**：
1. **门控计算**：U_{k,C} = σ(Gate_1(Z_C))，V_{m,C} = σ(Gate_2(Z_C))
2. **专家计算**：所有专家都激活（Dense）
   - B 个共享专家（所有 token、所有样本共用）
   - n×c 个细粒度专家（切分自基础专家）
3. **加权聚合**：h_l^C = Σ U_{k,C}·FFN_k + Σ V_{m,C}·FFN_m

**关键概念**：
- **细粒度专家切分**：将大专家切分为小份，增加组合灵活性，总参数量不变
- **共享专家隔离**：稳定的全局特征（如长期用户画像）由共享专家捕获，避免冗余学习

**关键公式**：
- 公式 11：h_l^C = Σ_{k=1}^B U_{k,C}·FFN_k + Σ_{m=1}^{n·c} V_{m,C}·FFN_m
- 公式 12：U_{k,C} = σ(Gate_1(D_l^C))，V_{m,C} = σ(Gate_2(D_l^C))

**与 RankMixer 的关键差异**：
- RankMixer：Per-token 独立专家，ReLU 动态稀疏，DTSI 训练策略
- MTmixAtt：全局共享专家，Sigmoid 始终密集，所有专家始终激活

### 3.5 第四层：Scenario Sparse MoE（场景稀疏专家）

**设计动机**：美团多场景（首页、团购、外卖等）需要平衡共享与隔离

**数据流转**：
1. **场景 token 拼接**：D_l^C ‖ D_l^s（物品 token ‖ 场景 token）
2. **共享专家门控**：Ω_{k,C} = σ(Gate_3([D_l^C ‖ D_l^s]))
3. **场景专家路由**：I_{k,C} = Gate_4([...]) + W·𝟙[k=k*]
   - k*：当前样本所属场景 ID
   - W > 0：手动设置的 bonus，强制激活当前场景专家
4. **Top-k 选择 + Mask**：保留 Top-k 最高分数专家
5. **Sigmoid 归一化**：Ψ_{k,C} = σ(Ĩ_{k,C})

**关键公式**：
- 公式 13：Ω_{k,C} = σ(Gate_3(D_l^C ‖ D_l^s))
- 公式 14：I_{k,C} = Gate_4([...]) + W·𝟙[k=k*]
- 公式 17：Υ_{k,C} = Ψ_{k,C} + Ω_{k,C}（如果 k 在 Top-k 中）

**关键设计**：手动 bonus W 确保当前场景专家一定被选中，同时通过 Top-k 选择其他相关场景专家实现知识迁移

### 3.6 输出层与预测头

**Token 聚合**：
- 输入：N 个 token 的表示 [h_1, h_2, ..., h_N]
- Mean Pooling：h_global = mean(h_1, h_2, ..., h_N)

**场景特定预测**：
- 主场景：y = DNN(h_global)
- 其他场景：y = DNN(h_global) + MLoRA_s(h_global)
  - MLoRA：低秩适配器，W_down ∈ R^{d×r}，W_up ∈ R^{r×d}，r << d

**损失函数**：
- 公式 18：L = -1/K Σ_{s=1}^K E[ y log ŷ + (1-y) log(1-ŷ) ]
- 对所有场景的损失取平均，端到端联合训练

## 四、关键创新点（技术贡献）

| 创新 | 解决的问题 | 核心价值 |
|------|-----------|---------|
| **AutoToken** | 人工特征分组的主观性 | 数据驱动自动分组，跨场景自适应 |
| **可学习混合矩阵** | 固定重排表达能力受限 | 更强的特征交互能力，正交初始化稳定训练 |
| **共享密集 MoE** | Per-token 专家孤立无共享 | 全局知识共享，减少冗余，细粒度切分提升灵活性 |
| **场景稀疏 MoE** | 多场景建模困难 | 共享专家 + 场景专家的混合，平衡迁移与隔离 |
| **MLoRA 适配** | 新场景适配成本高 | 低秩适配器实现参数高效的新场景扩展 |

## 五、实验结果

### 5.1 与 RankMixer 的直接对比（美团 TRec 数据集）
| 模型 | CTR AUC | CTR GAUC | CTCVR AUC | CTCVR GAUC | 参数量 |
|------|---------|----------|-----------|------------|--------|
| RankMixer | 0.7781 | 0.6920 | 0.8832 | 0.7658 | 14M |
| **MTmixAtt-15M** | **0.7792** | **0.6931** | **0.8840** | **0.7682** | 15M |
| **MTmixAtt-1B** | **0.7811** | **0.6963** | **0.8858** | **0.7714** | 1B |

**注意**：数据集、任务、场景设置均不同，直接对比需谨慎

### 5.2 消融实验关键发现

**AutoToken 消融（图 4）**：
- G1 随机分组 < G3 人工分组 ≈ G2 AutoToken

**混合矩阵初始化消融（图 4）**：

## 六、优缺点与局限性

### 6.1 优势
| 优势 | 说明 |
|------|------|
| **全自动特征分组** | AutoToken 消除人工特征工程，跨场景自适应 |
| **多场景支持** | 显式建模多场景，MLoRA 快速适配新场景 |
| **表达能力更强** | 可学习矩阵比固定重排更灵活 |
| **端到端训练** | 统一框架，无需 DTSI 等复杂训练策略 |
| **业务效果显著** | 首页 Payment PV +3.62%，GTV +2.54% |

### 6.2 局限性与待验证问题
| 局限 | 说明 |
|------|------|
| **硬件效率未报告** | 未报告 MFU、GPU 利用率、推理延迟等关键指标 |
| **计算开销** | 可学习矩阵和 AutoToken 引入额外计算，具体影响未知 |
| **与 RankMixer 对比的公平性** | 数据集、任务、场景设置均不同，结论需谨慎解读 |
| **超参数敏感性** | 共享专家数 B、细粒度切分数 c、Top-k 的 k 等需要调参 |
| **收敛速度** | AutoToken 需同时学习分组和模型，早期收敛可能更慢 |
| **S 矩阵可解释性** | 学习到的分组是否能对应真实语义？ |

## 八、核心启示

1. **自动化是工业落地的趋势**：从 RankMixer 的人工分组到 MTmixAtt 的 AutoToken，减少人工干预是方向
2. **多场景建模需要显式设计**：简单的共享或独立的极端都不可取，共享+隔离的混合更优
3. **表达能力与效率需要权衡**：MTmixAtt 用一定效率换取自动化和灵活性，适用于多场景复杂业务
4. **正交初始化的小技巧**：对于残差连接中的可学习变换，正交初始化（≈ I）能稳定训练
